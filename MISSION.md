# Mission: Godot Combat Systems Architecture

## Why
Use the cyber-arpg sandbox to learn practical Godot 4 .NET/C# architecture and game systems design by tracing, explaining, and incrementally extending real combat code.

## Success looks like
- Explain one Kinetic Burst shot from input to hit feedback using concrete scene and script references.
- Distinguish Godot scene composition, exported C# properties, and runtime node behavior.
- Review small combat-system changes for bugs, tradeoffs, missing validation, and learning value.

## Constraints
- Teach in small chunks with explain-back checkpoints and short knowledge checks.
- Keep work inside Milestone 2 unless the user explicitly changes scope.
- Do not modify project files unless the user explicitly asks for implementation or handoff work.

## Out of scope
- Enemy AI, wave spawning, enemy archetypes, loot, status effects, and death lifecycle systems.
- Generalized VFX frameworks, event buses, or data-driven ability architecture before the user asks for them.
